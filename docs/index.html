Requerimientos funcionales

Una visión general

requerimiento uno

El sistema deberá permitir donaciones mediante criptomonedas digitales y otras monedas tradicionales, generando al donante más posibilidades al momento de donar.

criterios de aceptación

Si un usuario requiere hacer una donación, entonces el sistema deberá obligarle a registrarse para poder realizar la transacción.

Ejecución: este acceso otorga al usuario el privilegio de ejecutar transacciones, opciones o módulos.

requerimiento dos

El sistema debe permitir la gestión de donantes y organizaciones con el fin de garantizar los roles adecuados.

criterios de aceptación

Cuando un usuario nuevo que no esté registrado ingrese al sistema, este deberá permitirle acceder todas las funciones del sistema pero no deberá permitirle realizar donaciones.

En el momento del registro, el sistema debe permitir al usuario registrarse mediante la cuenta de correo de gmail, o facebook para facilitar el proceso de registro.

En el momento que desee un usuario debe poder seguir el flujo de un proyecto, como por ejemplo, acceder, ver su descripción, ver los fondos que ha recibido, cuanto se ha gastado, cuánto conserva, las transacciones que ha realizado con los fondos y fotos de las actividades(opcional), con el fin de generar buenas sensaciones en los donantes.

El sistema deberá exigir la validación de la organización previo a crear un proyecto, para garantizar que sean las organizaciones quienes creen los proyectos.

requerimiento tres

El sistema deberá permitir la gestión de causas que actuarán como representación digital de las obras sociales.

criterios de aceptación

La duración de un proyecto no debe ser mayor a un año. Dado lo anterior, cuando se cumpla el tiempo, el proyecto dejará de estar en modo implementación y pasará a modo histórico.

En cualquier situación los usuarios deberán tener la opción de compartir los proyectos en sus redes sociales para cautivar la atención de más donantes.

Al momento de crear un proyecto nuevo, la entidad debe establecer una meta económica, la cual puede ser superada, pero sirve de referencia para ver el crecimiento de las donaciones.

requerimiento cuatro

El sistema debe permitir la visualización del histórico de operaciones al interior de las causas (proyectos), con el fin de generar empatía con el donante.

criterios de aceptación

Lectura: el usuario puede únicamente leer o visualizar la información pero no puede alterarla. Debe considerarse que la información puede ser copiada o impresa.
